Curso de Git Hup

Para tabajar un proyecto en git se utilizan tres estados

Working Directory: Es el directorio donde se trabaja con todos los archivos.
Stating Área : Es donde se agregar todos los archivos que se prepara para el guardado (listado de múltiples cambios).
Repository : Almacenamiento final de la versión.

Comandos basicos de consola Git Bash

  • ls: Permite listar el cotenido del directorio
  • pw: Permite conocer el directorio actual dode estamos ubicado
  • cd: Permite ubicarnos en un directorio.

Para para adminisras un proyecto en Git

  1. Iniciamos el proyecto con git para controlar las versiones

    2. Para iniciar el proyecto en git utilizamos el comando git init y damos enter.

  2. Conocer el estado de los archivos

    3. Para conocer el estado de los archivos que estmos siguiendo, se utiliza el comando git status y damos enter.

  3. Cofigurar el usuario global

    4. Se cofigura el correo del usuario que realizara los cambiosgit config –global user.email wicho80@gmail.com y damos enter.

    Se cofigura el nombre del usuario que realizara los cambiosgit config –global user.name Wilson y damos enter.

  4. Agragar archivos al entorno de trabajo (nuevos cambios)

    5. Para agregar archivos al area del trabajo se utiiza el acomando git add "nombre_archivo" y damos enter.

    Podemos agragar todos los archvios con el siguiente comando git add . y damos enter.

  5. Crear punto de partida en el repositorio (guradar cambios)

    6. para crear un punto de partida (cambios realizados) se utiliza el comando git commit y damos enter, luego presionamos la tecla i para isertar el comenterio de los cambios realizado, presionamos escape :wq y damos enter.

    Tambien se puede utilizar el siguiente comando git commit –m “texto de modificación”

  6. Crear todos los cambios realizados en el proyecto

    7. Para conocer el historial de los cambios realizados se utiliza el comando git log y damos enter.

  7. Volver a un estado anterior

    8. Para retroceder los cambios realizados se utliza el comando git checkout -- nombre del archivo y damos enter.

  8. Conocer las diferencias de los cambios realizados

    9. Para concer las deferencias de los cambios realizados en los archivos se utliza el comando git dif nombre del archivo y damos enter.

  9. Ignorar archivos

    10. Para ignorar los archivos en el control de versiones se crea una archivo con el nombre .gitignore, y en el agregamos las carpetas o archivos que deseamos ingnorar

  10. Crear version alternativa del codigo (rama)

    11. Primero debemos comocer en que version estamos ubicado con el comando git branch

    Para crear una versión alternativa se utiliza el comando git branch nombre_version

    Para seleccionar una version se utiliza el comando git checkout nombre_version

git remote add origin https://github.com/wilsonmartinezs/curso_git.git
git push -u origin master

Clonar repositorio de git hup en el equipo local
